基础篇

1. 哲学差异与语言概览

C# 和 C++ 都是 C 家族的强类型语言，但它们的设计哲学有本质区别。理解这些差异是学好 C++ 的第一步。

1.1 设计哲学对比

维度C#C++
抽象层次高级抽象，依赖 CLR 运行时零成本抽象，直接映射到硬件
性能理念"足够快"，JIT 编译优化"你不使用的东西不应付出代价"
内存管理GC 自动回收手动管理 + RAII + 智能指针
编译目标IL 中间代码 → JIT直接编译为原生机器码
安全模型内存安全（默认），类型安全信任程序员，允许底层操作
标准库.NET BCL（极其庞大）STL（精简高效）+ 第三方库
向后兼容版本迭代较平滑极度重视向后兼容（C 兼容）

1.2 "零成本抽象"的含义

这是 C++ 最核心的设计原则。在 C# 中，使用高级特性（如 LINQ、反射、装箱）往往有运行时开销。C++ 的设计目标是：你使用的抽象（类、模板、迭代器等）在编译后应和手写的底层代码一样高效。

C++
// std::sort 在编译时展开为针对具体类型的高效排序代码
// 没有虚函数调用、没有装箱、没有委托开销
#include <algorithm>
#include <vector>

std::vector<int> nums = {5, 2, 8, 1, 9};
std::sort(nums.begin(), nums.end());  // 编译后等价于手写的 int 排序

C++ 每三年发布一次新标准。作为 C# 开发者，你主要会接触到现代 C++（C++11 及之后）：

标准关键特性类比 C#

建议

作为新项目，建议直接使用 C++17 或 C++20 标准。它们提供了大量现代语法糖，让你从 C# 过渡时更加顺畅。

2. 编译模型与项目结构

这是 C# 开发者学习 C++ 时遇到的第一个根本性差异。C++ 没有 CLR，没有统一的程序集概念，编译模型完全不同。

2.1 编译流程

C# 的编译流程是：.cs 文件 → 编译器 → IL 程序集 → JIT 运行时编译为机器码。

C++ 的编译流程分为三步：

  1. 预处理（Preprocessing）：处理 #include、#define、#ifdef 等指令，展开头文件。
  2. 编译（Compilation）：每个 .cpp 文件独立编译为 .obj（Windows）或 .o（Linux）目标文件。
  3. 链接（Linking）：将所有目标文件和库文件链接为最终的可执行文件或动态库。
Shell
# 编译单个源文件
g++ -std=c++20 -Wall -O2 main.cpp -o myapp

# 编译多个源文件
g++ -std=c++20 -Wall -O2 main.cpp math.cpp logger.cpp -o myapp

# 分步编译：先编译为目标文件，再链接
g++ -c math.cpp -o math.o
g++ -c logger.cpp -o logger.o
g++ main.cpp math.o logger.o -o myapp

2.2 头文件与源文件

C++ 项目的代码组织方式与 C# 截然不同。C# 中每个类通常放在一个 .cs 文件中，编译器自动处理所有依赖。C++ 中需要将接口声明和实现分开：

概念C#C++
接口声明直接在 .cs 文件中头文件 (.h / .hpp)
实现直接在 .cs 文件中源文件 (.cpp)
引入其他文件using 命名空间#include 头文件（文本替换）
前向声明一般不需要常用，减少编译依赖
模块系统程序集 + usingC++20 Modules（尚不普及）
C#
// Player.cs - 一个文件搞定一切
namespace Game
{
    public class Player
    {
        public string Name { get; set; }
        public int Health { get; set; } = 100;

        public void TakeDamage(int amount)
        {
            Health -= amount;
        }
    }
}
C++
// Player.h - 头文件（声明）
#pragma once
#include <string>

class Player {
public:
    std::string name;
    int health = 100;

    void TakeDamage(int amount);
};

// Player.cpp - 源文件（实现）
#include "Player.h"

void Player::TakeDamage(int amount) {
    health -= amount;
}
注意

#include 是文本替换，不是 C# 的 using。#include "Player.h" 会把 Player.h 的全部内容复制粘贴到当前位置。这就是为什么需要头文件保护（#pragma once）来防止重复包含。

2.3 翻译单元与 ODR

每个 .cpp 文件加上它包含的所有头文件构成一个翻译单元（Translation Unit）。C++ 有一个关键规则叫 ODR（One Definition Rule，单一定义规则）：任何变量、函数、类在整个程序中只能有一个定义。违反 ODR 会导致链接错误。

内联函数例外

标记为 inline 的函数和模板可以放在头文件中，它们不受 ODR 限制。C++17 的 inline 变量也是如此。

3.1 基础类型对照

C#C++说明

uintunsigned intC++ 的无符号类型更常用
long (64 位)long longC++ 的 long 在 Windows 上只有 32 位！
byteuint8_t推荐用 <cstdint> 固定宽度类型

C# 明确保证 int 是 32 位、long 是 64 位。C++ 标准只规定了最小范围，实际大小取决于平台和编译器。在跨平台代码中，强烈建议使用 <cstdint> 中的固定宽度类型：int32_t、int64_t、uint8_t 等。

3.2 值类型 vs 引用类型

这是从 C# 到 C++ 最关键的思维转换之一。C# 严格区分值类型（struct）和引用类型（class）。在 C++ 中，所有类型都可以是值类型，取决于你如何使用它。

C#
// class = 引用类型（在堆上分配）
var p1 = new Player();
var p2 = p1;  // p1 和 p2 指向同一个对象

// struct = 值类型（在栈上分配）
var v1 = new Vector2(1, 2);
var v2 = v1;  // v2 是 v1 的独立副本
C++
// 栈上创建 - 值语义
Player p1;
Player p2 = p1;  // p2 是 p1 的独立副本！

// 堆上创建 - 需要手动管理
auto* p3 = new Player();
auto* p4 = p3;   // p3 和 p4 指向同一对象
delete p3;        // 必须手动释放

// 现代 C++：用智能指针
auto p5 = std::make_shared<Player>();
auto p6 = p5;    // 引用计数 +1，类似 C# 引用
核心区别

在 C# 中，class 和 struct 的区别决定了内存分配方式。在 C++ 中，class 和 struct 的唯一区别是默认访问权限（class 默认 private，struct 默认 public）。对象是栈分配还是堆分配取决于你如何创建它，而非类型定义。

3.3 const 正确性

C++ 的 const 比 C# 的 readonly 和 const 强大得多。const 在 C++ 中是一种"契约"，贯穿整个类型系统。

C++
// const 变量
const int MAX_HP = 100;       // 编译期常量
constexpr int AREA = 10 * 20;  // 强制编译期计算

// const 成员函数 - 承诺不修改对象
class Player {
    int health;
public:
    int GetHealth() const { return health; }
    void TakeDamage(int dmg);
};

// const 引用 - 避免拷贝又不允许修改
void PrintPlayer(const Player& p) {
    std::cout << p.GetHealth();  // OK
    // p.TakeDamage(10);        // 编译错误！
}
最佳实践

养成"const 正确性"的习惯：如果成员函数不修改对象，就标记为 const；如果函数参数不需要修改，就用 const 引用传递。

3.4 enum 与 enum class

C#
enum Direction
{
    Up, Down, Left, Right
}
int d = (int)Direction.Up;
C++
enum class Direction {
    Up, Down, Left, Right
};
auto d = Direction::Up;
int i = static_cast<int>(d);

始终使用 enum class 而非旧式的 enum。enum class 提供了作用域限定和类型安全。

4. 指针与引用

这是 C# 开发者转向 C++ 时必须跨越的最大障碍之一。C# 中指针只在 unsafe 代码中出现，而 C++ 中指针和引用无处不在。

4.1 理解内存地址

C++
int x = 42;

int* ptr = &x;       // ptr 存储了 x 的内存地址
std::cout << ptr;     // 输出地址，如 0x7fff1234
std::cout << *ptr;    // 解引用：输出 42

*ptr = 100;             // 通过指针修改 x 的值
std::cout << x;       // 输出 100

4.2 指针 vs 引用

特性指针 (T*)引用 (T&)
能否为空可以（nullptr）不能（必须绑定到有效对象）
能否重新绑定可以指向其他对象不能，一旦绑定就不可变
语法需要 * 解引用直接使用，和原变量一样
多级间接支持（T**）不支持
何时使用可选参数、动态分配、底层操作函数参数、避免拷贝（最常用）
C++
void Example() {
    std::string name = "Alice";

    std::string& ref = name;      // 引用 - 别名
    ref = "Bob";                    // name 也变成 "Bob"

    const std::string& cref = name; // const 引用 - 只读

    std::string* p = &name;       // 指针 - 可以为空
    *p = "Charlie";
    p = nullptr;
}

4.3 函数参数传递方式

方式C#C++说明
按值传递void Foo(int x)void foo(int x)拷贝，适合小类型
引用传递void Foo(ref int x)void foo(int& x)不拷贝，可修改原值
只读引用void Foo(in int x)void foo(const int& x)不拷贝，不可修改
指针传递unsafe void Foo(int* x)void foo(int* x)可以为 null
悬垂引用

不要返回局部变量的引用！局部变量在函数结束后销毁，引用变成悬垂引用（dangling reference），行为未定义。

C++
// 危险！不要这样做！
std::string& GetName() {
    std::string name = "Alice";
    return name;  // 灾难！name 在函数结束后被销毁
}

// 正确做法：按值返回（编译器会做 RVO 优化）
std::string GetName() {
    std::string name = "Alice";
    return name;  // OK：编译器优化后不会真正拷贝
}

5. 内存管理

内存管理是 C# 和 C++ 之间最大的鸿沟。C# 有 GC 自动管理内存；C++ 则需要你理解并手动控制内存的分配与释放——不过现代 C++ 提供了 RAII 和智能指针来大幅简化这件事。

5.1 栈 vs 堆

特性栈 (Stack)堆 (Heap)
分配方式自动（变量声明时）手动（new 或智能指针）
释放方式自动（离开作用域时）手动 delete 或智能指针
速度极快（移动栈指针）较慢（搜索可用内存块）
大小限制有限（通常 1-8 MB）很大（受系统内存限制）
生命周期作用域结束即销毁直到手动释放或程序结束
C++
void Example() {
    // === 栈分配（推荐，默认首选） ===
    int x = 42;
    Player player;
    std::vector<int> nums(100);

    // === 智能指针（现代 C++ 推荐） ===
    auto sp = std::make_unique<Player>();
    auto sh = std::make_shared<Player>();
} // 所有变量在这里自动销毁/释放

5.2 RAII — C++ 最重要的惯用法

RAII（Resource Acquisition Is Initialization，资源获取即初始化）是 C++ 的核心理念：把资源的生命周期绑定到对象的生命周期上。对象在栈上创建时获取资源，离开作用域时自动调用析构函数释放资源。

这相当于 C# 的 using 语句 / IDisposable 模式，但不需要你显式写 using，完全自动化。

C#
// C# 需要显式 using 或 try/finally
using (var file = File.OpenRead("data.txt"))
{
    // 使用 file
} // using 结束时自动 Dispose
C++
// C++ RAII：析构函数自动清理
{
    std::ifstream file("data.txt");
    // 使用 file
} // file 离开作用域，析构函数自动关闭

{
    auto ptr = std::make_unique<Player>();
    // 使用 ptr
} // unique_ptr 析构时自动 delete
现代 C++ 几乎不需要裸 new/delete

在现代 C++ 中，应优先使用智能指针和容器来管理内存。直接使用 new/delete 是遗留风格，容易出错。

6. 函数深入

6.1 函数重载与默认参数

C++ 和 C# 都支持函数重载。C++ 的默认参数有一些独特规则：默认参数必须从右向左连续提供，且默认值写在声明中（头文件），C++ 不支持命名参数。

C#
void Spawn(string name,
    int health = 100,
    float scale = 1.0f) { }

// 可以用命名参数跳过中间参数
Spawn("Boss", scale: 2.5f);
C++
void Spawn(const std::string& name,
    int health = 100,
    float scale = 1.0f);

Spawn("Boss");           // OK
Spawn("Boss", 200);       // OK
// 不能跳过中间参数！

6.2 函数指针与 std::function

C# 有 delegate、Action、Func。C++ 对应的是函数指针和 std::function。

C#
Func<int, int, int> add =
    (a, b) => a + b;
int result = add(3, 4);  // 7
C++
std::function<int(int, int)> add =
    [](int a, int b) { return a + b; };
int result = add(3, 4);  // 7

7. 面向对象编程

7.1 类的基础

C++ 的类与 C# 有很多相似之处，但语法差异显著。最大的不同是没有属性（Property）语法和接口（Interface）关键字。

C#
public class Animal
{
    public string Name { get; set; }
    public virtual int Age { get; set; }

    public Animal(string name) {
        Name = name;
    }

    public virtual void Speak() {
        Console.WriteLine("...");
    }
}
C++
class Animal {
public:
    std::string name;
    int age;

    Animal(const std::string& n)
        : name(n), age(0) {}

    virtual void Speak() const {
        std::cout << "...";
    }

    virtual ~Animal() = default;
};

7.2 构造函数与析构函数

C++ 有多种构造函数，比 C# 丰富得多：

C++
class Player {
    std::string name;
    int health;
    std::vector<int> scores;

public:
    // 1. 默认构造函数
    Player() : name("Unknown"), health(100) {}

    // 2. 参数化构造函数（初始化列表，推荐！）
    Player(const std::string& n, int hp)
        : name(n), health(hp) {}

    // 3. 委托构造函数
    Player(const std::string& n)
        : Player(n, 100) {}

    // 4. 拷贝构造函数
    Player(const Player& other);

    // 5. 移动构造函数（C++11）
    Player(Player&& other) noexcept;

    // 6. 析构函数 - C# 没有对应物！
    ~Player() { /* 清理资源 */ }
};
Rule of Five / Rule of Zero

如果你需要自定义析构、拷贝构造、拷贝赋值、移动构造、移动赋值中的任何一个，通常需要全部五个都自定义（Rule of Five）。但如果你的类只使用 RAII 类型（如 std::string、std::vector、智能指针），则一个都不需要写（Rule of Zero）——编译器自动生成的版本就是正确的。

7.3 继承与多态

C#
public class Dog : Animal
{
    public override void Speak() {
        Console.WriteLine("Woof!");
    }
}

Animal a = new Dog();
a.Speak();  // "Woof!"
C++
class Dog : public Animal {
public:
    void Speak() const override {
        std::cout << "Woof!";
    }
};

// 多态必须通过指针或引用！
Dog dog;
Animal& a = dog;
a.Speak();  // "Woof!" - 正确

// 注意：按值传递会"切片"！
Animal b = dog;  // 切片！
b.Speak();  // "..." - 丢失了 Dog
对象切片 (Object Slicing)

这是 C# 开发者最容易踩的坑。在 C++ 中，如果按值赋值（不是引用或指针），派生类会被"切片"为基类部分。要实现多态，必须通过指针或引用。

7.4 纯虚函数与抽象类

C++ 没有 interface 关键字，用纯虚函数（= 0）实现接口效果：

C++
class IDrawable {
public:
    virtual void Draw() = 0;       // 纯虚函数
    virtual int GetLayer() = 0;
    virtual ~IDrawable() = default; // 虚析构函数必须有！
};

class Sprite : public IDrawable {
public:
    void Draw() override { /* 绘制 */ }
    int GetLayer() override { return 1; }
};
虚析构函数

如果一个类有任何虚函数，一定要给它声明虚析构函数。否则通过基类指针 delete 派生类对象时，派生类的析构函数不会被调用，导致资源泄漏。

7.5 多重继承

C# 只支持单继承，C++ 支持多重继承。实际项目中，多重继承通常只用于"接口 + 一个实现类"的模式。

C++
class Sprite : public GameObject,   // 基类实现
               public IDrawable,    // 接口
               public ISerializable // 接口
{
    // 实现所有纯虚函数
};

8. 运算符重载

C++ 的运算符重载比 C# 灵活得多。C++ 允许重载几乎所有运算符，且可以自由定义返回类型。

C++
struct Vec2 {
    float x, y;

    Vec2 operator+(const Vec2& other) const {
        return {x + other.x, y + other.y};
    }
    Vec2 operator*(float scalar) const {
        return {x * scalar, y * scalar};
    }
    bool operator==(const Vec2& other) const {
        return x == other.x && y == other.y;
    }
    float& operator[](int index) {
        return index == 0 ? x : y;
    }
    friend std::ostream& operator<<(std::ostream& os, const Vec2& v) {
        return os << "(" << v.x << ", " << v.y << ")";
    }
};

Vec2 a{1, 2}, b{3, 4};
auto c = a + b * 2.0f;
std::cout << c;  // "(7, 10)"

9. 异常处理

9.1 基本语法对比

C#
try {
    throw new ArgumentException("Bad!");
}
catch (Exception ex) {
    Console.WriteLine(ex.Message);
}
finally {
    Cleanup();
}
C++
try {
    throw std::runtime_error("Bad!");
}
catch (const std::exception& e) {
    std::cout << e.what();
}
// 没有 finally！用 RAII 代替

9.2 关键差异

特性C#C++
异常基类System.Exceptionstd::exception
finally 块有没有（用 RAII 代替）
throw 的对象只能是 Exception 子类任何类型
noexcept无对应noexcept 标记不抛异常
性能哲学异常较常用异常开销大，不建议用于控制流
C++ 异常使用哲学

在 C++ 社区中，异常的使用比 C# 保守得多。很多项目（如游戏引擎、嵌入式系统）完全禁用异常。替代方案包括：返回错误码、std::optional、std::expected（C++23）。

    file << "some data";
    // 即使抛异常，file 析构时也会自动关闭
}

// 自定义 ScopeGuard（类似 defer / finally）
struct ScopeGuard {
    std::function<void()> cleanup;
    ~ScopeGuard() { cleanup(); }
};

{
    AcquireLock();
    ScopeGuard guard{[] { ReleaseLock(); }};
    // 即使抛异常，锁也会被释放
}

13. 智能指针与 RAII

13.1 三种智能指针

类型所有权模型类比 C#何时使用
unique_ptr独占所有权IDisposable 独占资源默认首选
shared_ptr共享所有权（引用计数）普通引用多个持有者
weak_ptr不拥有，观察WeakReference打破循环引用
C++
#include <memory>

// unique_ptr — 独占所有权
{
    auto p = std::make_unique<Player>("Alice", 100);
    p->Speak();
    auto p2 = std::move(p);  // 转移所有权
    // auto p3 = p;            // 编译错误！不能拷贝
} // 自动 delete

// shared_ptr — 引用计数
{
    auto s1 = std::make_shared<Player>("Bob", 80);
    {
        auto s2 = s1;  // 引用计数 = 2
    } // s2 销毁，引用计数 = 1
} // s1 销毁，引用计数 = 0，delete

// weak_ptr — 打破循环引用
{
    auto shared = std::make_shared<Player>("C", 90);
    std::weak_ptr<Player> weak = shared;
    if (auto locked = weak.lock()) {
        locked->Speak();  // 对象还活着
    }
}

13.2 Move 语义

Move 语义是 C++11 引入的核心特性，C# 没有直接对应物。核心思想：对于即将销毁的临时对象，直接"偷走"它的资源而非深拷贝。

C++
std::vector<int> CreateLargeVector() {
    std::vector<int> v(1000000);
    return v;  // 不会拷贝！自动 move 或 RVO
}

std::vector<int> data = CreateLargeVector();  // 零拷贝

std::string s1 = "Hello World This Is Long";
std::string s2 = std::move(s1);  // s2 偷走了 s1 的缓冲区
                                  // s1 现在是有效但未定义的状态

17. 常见陷阱与最佳实践

17.1 C# 开发者最容易犯的 10 个错误

1. 忘记 #include 头文件

C# 的 using 只是引入命名空间。C++ 的 #include 是真正地把文件内容复制过来。忘记 #include 会导致编译错误，但错误信息可能很晦涩。

2. 对象切片

如前所述，多态必须通过引用或指针。按值传递派生类对象会丢失派生类部分。

3. 忘记虚析构函数

有虚函数的类必须有虚析构函数。否则通过基类指针 delete 时，派生类析构函数不会被调用。

4. 悬垂指针 / 悬垂引用

返回局部变量的引用、使用已释放的内存、容器 resize 后迭代器失效——这些都会导致未定义行为。

5. 混用 signed 和 unsigned

std::vector::size() 返回 size_t（无符号），与 int 比较可能产生意外结果。

6. 忘记初始化变量

C++ 的内置类型（int, float, 指针等）不会自动初始化为零！必须显式初始化。

C++
int x;        // 未初始化！值是不确定的垃圾值
int y{};      // 零初始化，值为 0（推荐）
int z = 0;    // 显式初始化为 0

int* p;       // 未初始化！危险的野指针
int* q = nullptr;  // 安全

7. 头文件循环依赖

两个头文件互相 #include 会导致编译失败。解决方案：使用前向声明（forward declaration）代替不必要的 #include。

8. 忽略 const 正确性

在 C++ 社区中，const 正确性是代码质量的重要标志。不写 const 会传播：一个非 const 方法会导致所有调用它的地方都无法使用 const。

9. 使用裸 new/delete

现代 C++ 几乎总是应该使用智能指针。裸 new/delete 是内存泄漏的温床。

10. 忽略移动语义

不理解 move 语义意味着错过大量性能优化机会，也意味着无法正确使用某些现代 C++ 特性。
