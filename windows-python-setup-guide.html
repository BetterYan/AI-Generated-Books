Windows Python 深度学习环境安装指南

为 PyTorch 模型转换、ONNX 导出、YOLO 原型验证打造的完整工具链 setup 手册

PyTorch ONNX YOLO CUDA Windows 10/11
概览 安装 Python 虚拟环境 PyTorch ONNX YOLO 工作流 背景故事 速查表

为什么要用这套环境？

你的核心需求可以归纳为三类：

  • 模型转换：用 PyTorch 训练或加载模型，导出为 ONNX 格式，供其他框架或边缘设备部署。
  • 模型验证：下载社区预训练权重，快速跑通推理，确认模型能力是否符合预期。
  • 原型验证：用 YOLO 系列做目标检测原型，验证数据、调参、评估指标。

这套指南不追求"最全"，而是追求"最顺"——每一步都给出Windows 下最稳的安装路径，以及出问题时最常见的排查方法。

1安装 Python

1.1 下载官方 Python

推荐从 python.org 下载最新稳定版（当前建议 3.11.x）。

为什么不是 3.12？ 部分 PyTorch 的预览功能或第三方库对 3.12 的适配可能滞后。3.11 在性能与兼容性之间是目前的甜点版本。如果你的目标库已明确支持 3.12，也可以上。

安装时务必勾选：

  • Add python.exe to PATH（添加到环境变量）
  • Use admin privileges when installing py.exe

安装完成后，打开 PowerShell 或 CMD 验证：

python --version
pip --version

1.2 确认 pip 和基本工具

python -m pip install --upgrade pip setuptools wheel

这三个是基础中的基础：setuptools 负责包的构建，wheel 让安装预编译包时速度翻倍。

2创建虚拟环境

Python 的虚拟环境相当于给每个项目一个独立的"试剂瓶"，避免包版本冲突。这是好习惯，尤其在深度学习领域，不同模型可能对 numpy、opencv 版本要求不同。

2.1 创建并激活环境

# 在项目目录下创建一个名为 venv 的虚拟环境
python -m venv venv

# 激活（PowerShell）
venv\Scripts\Activate.ps1

# 激活（CMD）
venv\Scripts\activate.bat
PowerShell 执行策略问题
如果提示"无法加载脚本，因为在此系统上禁止运行脚本"，以管理员身份运行 PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

2.2 退出环境

deactivate

激活后，命令行提示符前会出现 (venv)，代表当前处于隔离环境中。

3安装 PyTorch

PyTorch 是整个工具链的核心。安装方式取决于你是否有 NVIDIA 独立显卡。

3.1 检查你的硬件

打开任务管理器 → 性能 → GPU，如果有 NVIDIA GPU，记住型号，然后去 CUDA GPU 列表 确认是否支持 CUDA。

3.2 有 NVIDIA 显卡（推荐 GPU 版）

先安装 NVIDIA 显卡驱动（GeForce Experience 或官网驱动即可），确保驱动较新。PyTorch 2.x 通常需要 CUDA 11.8 或 12.x。

去 pytorch.org 选择对应配置，生成命令。以下是目前最常用的两条：

# CUDA 12.4（较新的 40/50 系显卡推荐）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# CUDA 11.8（兼容性更广，30/20 系也适用）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

3.3 只有核显 / 纯 CPU

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

3.4 验证安装

python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU only')"

如果输出 True 和显卡名称，恭喜，GPU 加速就绪。

torchvision / torchaudio 是干什么的？
torchvision 提供图像预处理、常见模型（ResNet、MobileNet 等）和数据集工具；torchaudio 处理音频。如果你只做图像/模型转换，torchaudio 可以省略，但一起装也没坏处。

4安装 ONNX 工具链

ONNX（Open Neural Network Exchange）是模型跨框架迁移的桥梁。PyTorch 原生支持导出 ONNX，但你需要 surrounding tools 来验证、简化、优化导出的模型。

4.1 核心包

pip install onnx onnxruntime onnxscript
包名作用
onnxONNX 格式定义和基础工具，读写 .onnx 文件
onnxruntime微软开发的高性能推理引擎，支持 CPU/GPU，验证模型是否可正常推理
onnxscript用 Python 直接编写/转换 ONNX 图，调试时很有用

4.2 GPU 推理加速（可选）

如果你要在本机用 GPU 跑 ONNX 推理，安装 GPU 版 Runtime：

pip install onnxruntime-gpu
注意：onnxruntime 和 onnxruntime-gpu 不要同时安装，会冲突。如果已经装了 CPU 版，先卸载再装 GPU 版。

4.3 快速验证 ONNX 导出

import torch
import onnx

# 创建一个 dummy 模型
class DummyModel(torch.nn.Module):
    def forward(self, x):
        return x * 2

model = DummyModel()
dummy_input = torch.randn(1, 3, 224, 224)

# 导出
torch.onnx.export(
    model, dummy_input, "dummy.onnx",
    input_names=["input"], output_names=["output"],
    opset_version=17
)

# 验证
onnx_model = onnx.load("dummy.onnx")
onnx.checker.check_model(onnx_model)
print("ONNX 模型验证通过！")

# 用 onnxruntime 跑推理
import onnxruntime as ort
session = ort.InferenceSession("dummy.onnx")
outputs = session.run(None, {"input": dummy_input.numpy()})
print(outputs[0].shape)

4.4 ONNX 转换进阶工具（按需安装）

# 模型简化（去除冗余节点，减小体积）
pip install onnxsim

# 图优化工具
pip install onnxoptimizer

5安装 YOLO 系列

YOLO（You Only Look Once）是目标检测领域最具影响力的系列之一。当前社区主流是 Ultralytics 维护的 YOLOv8/v11，API 极其简洁。

5.1 安装 Ultralytics

pip install ultralytics

5.2 验证 YOLO 推理

from ultralytics import YOLO

# 自动下载 yolov8n.pt 权重到当前目录
model = YOLO("yolov8n.pt")

# 对一张图片做推理（可替换成你自己的图片路径）
results = model("https://ultralytics.com/images/bus.jpg")
results[0].show()

第一次运行会自动下载权重文件（约 6MB），保存在当前目录。

5.3 YOLO 导出 ONNX

这是你最可能用到的功能——把 YOLO 模型导出为 ONNX：

model = YOLO("yolov8n.pt")
model.export(format="onnx")  # 生成 yolov8n.onnx

5.4 你可能还需要 OpenCV

pip install opencv-python

YOLO 的结果可视化、摄像头取流、图片预处理都离不开 OpenCV。

opencv-python-headless 的区别
如果你只在服务器/无 GUI 环境运行，装 opencv-python-headless（没有 GUI 依赖，体积更小）。本地 Windows 开发建议装带 GUI 的 opencv-python。

6模型下载与验证工作流

以下是你日常会用到的典型工作流，建议收藏本节。

6.1 从 Hugging Face 下载模型

Hugging Face 是当前最大的预训练模型社区。安装其 Python 客户端：

pip install huggingface_hub

下载单个文件：

from huggingface_hub import hf_hub_download

model_path = hf_hub_download(
    repo_id="microsoft/resnet-50",
    filename="model.onnx"
)
print(model_path)

6.2 从 PyTorch Hub 加载

import torch

# 加载 torchvision 内置模型
model = torch.hub.load("pytorch/vision:v0.17.0", "resnet50", weights="IMAGENET1K_V2")
model.eval()

# 导出 ONNX
dummy = torch.randn(1, 3, 224, 224)
torch.onnx.export(model, dummy, "resnet50.onnx", opset_version=17)

6.3 模型能力快速验证清单

  1. 加载是否成功：模型文件能否正常加载，权重维度是否匹配。
  2. 前向推理是否通过：给定 dummy input，能否跑出结果且不报错。
  3. 输出形状是否符合预期：分类模型通常是 (batch, num_classes)，检测模型是 (batch, num_boxes, 6) 等。
  4. ONNX 导出是否成功：用 onnx.checker 验证图结构完整性。
  5. ONNX Runtime 能否推理：确保导出后的模型在推理引擎上跑得通。

依赖组件的背景故事

知其然，也知其所以然。了解这些工具的来龙去脉，有助于你在遇到问题时判断该往哪个方向排查。

1991 — Python
Guido van Rossum 的"圣诞礼物"
Python 诞生于荷兰的圣诞节假期。Guido 希望创造一种简洁、可读性强的语言，让程序员把精力花在解决问题上，而非语法细节上。30 多年后，Python 凭借 NumPy、SciPy 等科学计算生态，意外成为 AI 时代的第一语言。
2006 — CUDA
NVIDIA 的"通用计算"豪赌
CUDA 最初被游戏玩家和图形工作者忽视，但 NVIDIA 坚持让 GPU 能跑通用程序。当 2012 年 AlexNet 用两块 GTX 580 赢得 ImageNet 后，CUDA 成为深度学习的事实标准。没有 CUDA，现代 AI 的训练周期可能要延长数十倍。
2012 — AlexNet
深度学习的一声惊雷
虽然 AlexNet 本身不是工具，但它证明了 GPU + 深度卷积网络的可行性。从此，模型参数从百万级跃升至亿级，框架之争也拉开序幕：Caffe、Torch、Theano 相继登场。
2016 — PyTorch
Meta（Facebook）的"动态图"逆袭
基于古老的 Torch 框架，Meta 的 AI Research 团队推出了 PyTorch。它以动态计算图（define-by-run）为核心，调试时像写普通 Python 一样直观，迅速在学术界和工业界击败 TensorFlow 成为主流。PyTorch 2.0 引入的 torch.compile 更让它在性能上补足短板。
2017 — ONNX
微软与 Meta 的"停战协议"
深度学习框架割据（TensorFlow、PyTorch、Caffe2、MXNet...），模型互迁成本极高。微软与 Meta 联合发起 ONNX，试图定义一种开放的中间表示格式。如今 ONNX 已成为部署到移动端、边缘设备、甚至 Web 的标配中转格式。
2018 — ONNX Runtime
微软的推理引擎
光有格式不够，还需要一个高性能的"播放器"。ONNX Runtime 由微软开源，支持 CPU、CUDA、DirectML、OpenVINO 等多种后端。它不仅是验证工具，更是许多生产环境（包括 Azure、Office）的实际推理引擎。
2016 / 2023 — YOLO / Ultralytics
从学术论文到工程利器
Joseph Redmon 提出 YOLO 时，目标检测还是以两阶段方法（R-CNN 系列）为主。YOLO 的"单阶段、端到端"思路把速度提升了几个数量级。后来 Glenn Jocher 创办的 Ultralytics 公司推出 YOLOv5/v8/v11，用极其简单的 Python API 把目标检测变成几行代码的事，彻底降低了使用门槛。
2020 — Hugging Face
AI 界的"GitHub"
这个从聊天机器人创业失败转型而来的公司，意外建成了全球最大的预训练模型托管平台。Transformers 库几乎成了 NLP 的标准入口，而 Hugging Face Hub 也汇聚了大量视觉、语音、多模态模型，下载模型再也不是靠百度云盘了。

常见问题排查

pip 安装极慢或失败

换国内镜像源：

pip install xxx -i https://pypi.tuna.tsinghua.edu.cn/simple

或配置永久镜像：

pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
CUDA out of memory

减小 batch size，或改用 torch.no_grad() 上下文做推理。如果只是想验证模型，CPU 模式完全够用。

torch.cuda.is_available() 返回 False
  1. 检查 NVIDIA 驱动是否安装
  2. 检查 PyTorch 的 CUDA 版本是否与系统 CUDA 匹配
  3. 笔记本需确认没开在核显模式
Ultralytics 报错找不到模型

检查当前目录是否有 .pt 文件，或让 YOLO 自动下载：YOLO("yolov8n.pt") 第一次会自动拉取。

ONNX 导出后推理结果不对

重点检查 preprocessing（归一化、resize、通道顺序 BGR/RGB）和 postprocessing（NMS、置信度阈值）是否与训练时一致。ONNX 只负责"算"，不负责"预处理"。

PowerShell 激活环境报错

执行 Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser，或改用 CMD 的 activate.bat。

下一步建议

  1. 按顺序走完 Step 1~5，确保每个验证命令都能输出预期结果。
  2. 选定一个具体模型（比如 ResNet-50 或 YOLOv8n），完成"PyTorch 加载 → 推理验证 → ONNX 导出 → ONNX Runtime 再推理"的完整闭环。
  3. 把成功跑通的包列表导出为 requirements.txt，下次换机器时一键复现。
  4. 如果要做模型量化或部署到移动端，下一步可以了解 ONNX Runtime 的 INT8 量化和 TensorRT（NVIDIA 专用加速）。
